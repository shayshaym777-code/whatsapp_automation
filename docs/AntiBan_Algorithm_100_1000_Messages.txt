#### 3. How Each Account Sends After Distribution
After distribution, each account follows this sending protocol:

Per Account Sending Flow:
  1. Load Account Profile
     - Connect to WhatsApp Web with account's QR/token
     - Initialize WebView2 with account-specific fingerprint
     - Set proxy/VPN for this account's IP

  2. Message Queue Processing
     - Receive assigned messages (e.g., 20 messages for account)
     - Process one message at a time in SafeMode

  3. Per Message Execution:
     a. Select recipient from message queue
     b. Open chat window
     c. Apply behavioral randomization (typing simulation)
     d. Send message with randomized timing
     e. Wait for delivery confirmation
     f. Log result and update safety score

  4. Account-Level Controls:
     - Monitor delivery rate per account
     - Rotate device fingerprint every 10 messages
     - Change proxy IP every 10-20 messages
     - Force session break after 50 messages
     - Check daily limits (100 messages/account)

  5. Error Handling Per Account:
     - If message fails: retry with backoff
     - If account restricted: mark suspicious, switch to next
     - If proxy fails: rotate to new proxy
     - If session expires: re-authenticate

#### 8. Error Handling & Recovery Per Account

  - If error: Retry with exponential backoff (1s, 2s, 4s, 8s)
  - If 3 failures: Mark account suspicious, rotate to next account
  - If delivery rate < 95%: Add 2-3 second delays
  - If "Unusual activity": Immediate pause, investigate

Account Recovery:
  - Re-warm suspicious accounts
  - Change proxy/IP completely
  - Generate new device fingerprint
  - Wait 24-48 hours before reuse

  - פקח על אזהרות "פעילות לא רגילה"
  - האטה אוטומטית כשמתקרב לגבולות

ל-1000 הודעות: הפסקות סשן כל 100 הודעות, גבולות יומיים מורחבים.

#### 11. זיוף דפדפן ו-User-Agent
שילוב WebView2:
  - משתמש במנוע Chromium אמיתי
  - User-Agent אקראי לכל סשן
  - כותרות דפדפן משתנות

סיבוב User-Agent:
  מקורי: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
  וריאציות:
    • Firefox: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:89.0)
    • Safari: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)
    • Chrome נייד: Mozilla/5.0 (Linux; Android 10; SM-G973F)

מניפולציית כותרות בקשה:
  • User-Agent: אקראי לבקשה
  • Accept-Language: משתנה (en-US, he-IL וכו')
  • Accept-Encoding: דחיסה שונה
  • Referer: מזויף או מושמט
  • DNT (Do Not Track): משתנה

#### 12. שילוב חימום חשבונות
לפני שליחה בכמות גדולה:
  1. בחר 2-5 חשבונות יעד לחשבון השולח
  2. שלח 20-50 הודעות חימום על פני 2-3 ימים
  3. השתמש בדפוסי שיחה טבעיים
  4. בנה ציון אמון לחשבון
  5. פקח על התקדמות חימום

אלגוריתם חימום:
  - יום 1: 10-20 הודעות, 30-60 שניות עיכוב
  - יום 2: 15-30 הודעות, 20-45 שניות עיכוב
  - יום 3+: מוכן לקמפיינים

#### 13. טכניקות עקיפת זיהוי מתקדמות
שיטות זיהוי של וואטסאפ וצעדי נגד:

1. בדיקת מוניטין IP:
   זיהוי: "האם IP זה ידוע לפעילות בוט?"
   עקיפה: פרוקסי מגורים, סיבוב תכוף

2. טביעת אצבע מכשיר:
   זיהוי: MAC + שם מחשב + OS + מסך
   עקיפה: אקראי את כל הפרמטרים לכל סשן

3. ניתוח התנהגות:
   זיהוי: דפוסי תזמון הודעות
   עקיפה: עיכובים אקראיים, הקלדה דומה לאנושית

4. כותרות דפדפן:
   זיהוי: עקביות User-Agent
   עקיפה: סובב כותרות לבקשה

5. ניתוח תזמון:
   זיהוי: מרווחי עיכוב מדויקים
   עקיפה: רעידה + אקראיות

6. דפוס חשבון:
   זיהוי: כמות גדולה מחשבון יחיד
   עקיפה: חלוקה מרובה חשבונות

#### 14. מעקב וניתוח
מעקב בזמן אמת:
  - שיעור משלוח לחשבון
  - מעקב שיעור שגיאות
  - ביצועי פרוקסי
  - מגמות ציון בטיחות
  - סטטוס בריאות חשבון

רישום:
  - תוצאות שליחת הודעות
  - נתוני תזמון
  - פרטי שגיאות
  - שימוש בפרוקסי
  - טביעות אצבע מכשיר בשימוש

התראות:
  - שיעור משלוח < 95%
  - הגבלות חשבון
  - כשלונות פרוקסי
  - ירידות ציון בטיחות

#### 15. סיכום חישוב סופי
עיבוד 100 הודעות:
  → מבוזר על פני 5-10 חשבונות (10-20 הודעות כל אחד)
  → כל חשבון שולח עם SafeMode + פרוקסי
  → תזמון אקראי + פרופיל מכשיר ייחודי לחשבון
  → סיבוב פרוקסי כל 10-20 הודעות
  → הפסקות סשן + שינויי טביעת אצבע
  → תוצאה: שיעור משלוח 95%+, דפוס דומה לאנושי, סיכון באן נמוך

ל-1000 הודעות:
  → מורחב ל-20-50 חשבונות (20-50 הודעות כל אחד)
  → סיבובים תכופים יותר, הפסקות מורחבות
  → מעקב וטיפול שגיאות משופרים
  → תוצאה: שומר על שיעור משלוח גבוה עם נפח מוגבר
